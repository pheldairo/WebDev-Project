# Backend для WebDev-Project

Этот репозиторий содержит бэкенд часть проекта, разработанную на Django и Django REST Framework. Он предоставляет API для управления пользователями, комнатами, расписанием и данными университета, а также поддерживает аутентификацию на основе JWT.

## Описание

Бэкенд служит основой для веб-приложения, обеспечивая хранение данных, бизнес-логику и доступ к ним через RESTful API. Проект ориентирован на создание интерактивной платформы, которая может включать в себя функционал для организации учебного процесса, совместной работы или коммуникации.

## Технологии

*   **Python 3.x**
*   **Django**: Веб-фреймворк для быстрой разработки.
*   **Django REST Framework (DRF)**: Мощный и гибкий инструментарий для построения Web API.
*   **PostgreSQL**: Реляционная база данных.
*   **Django Simple JWT**: Реализация JSON Web Token (JWT) аутентификации для DRF.
*   **Django CORS Headers**: Управление Cross-Origin Resource Sharing (CORS).
## Функциональность

Бэкенд предоставляет следующие ключевые возможности, организованные по приложениям:

### 1. Пользователи (`backend.apps.users`)
*   **Регистрация пользователей**: Создание новых учетных записей.
*   **Аутентификация**: Вход в систему с использованием JWT-токенов.
*   **Управление профилем**: Просмотр и редактирование данных пользователя.
*   **Обновление/Отзыв токенов**: Механизмы для обновления истекших токенов доступа и отзыва токенов.

### 2. Комнаты (`backend.apps.rooms`)
*   **Создание комнат**: Пользователи могут создавать новые комнаты для различных целей (например, для встреч, занятий, чатов).
*   **Управление комнатами**: Просмотр списка комнат, деталей конкретной комнаты.
### 3. Расписание (`backend.apps.schedule`)
*   **Создание событий**: Добавление событий в расписание.
*   **Просмотр расписания**: Получение расписания по различным критериям (например, по пользователю, по комнате, по дате).
*   **Управление событиями**: Редактирование и удаление событий.

### 4. Университет (`backend.apps.university`)
*   **Управление данными университета**: Хранение и предоставление информации, связанной с университетом (например, факультеты, курсы, группы, преподаватели).
*   **Связь с расписанием и пользователями**: Интеграция данных университета с расписанием занятий и профилями пользователей (студентов/преподавателей).

### Общая функциональность
*   **RESTful API**: Все взаимодействия с бэкендом осуществляются через стандартизированный RESTful API.
*   **JWT Аутентификация**: Безопасная аутентификация пользователей с использованием JSON Web Tokens.
*   **CORS**: Настроен для обеспечения взаимодействия с фронтенд-приложениями, расположенными на других доменах.

## Установка и запуск

Для локальной установки и запуска проекта выполните следующие шаги:

### 1. Клонирование репозитория

```bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
cd WebDev-Project/backend
```

### 2. Создание и активация виртуального окружения

Рекомендуется использовать виртуальное окружение для изоляции зависимостей проекта.

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установка зависимостей

Установите все необходимые пакеты Python из файла `requirements.txt`. Предполагается, что `requirements.txt` находится в корневой директории проекта (`WebDev-Project`).

```bash
pip install -r ../requirements.txt
```

### 4. Настройка базы данных PostgreSQL

Убедитесь, что у вас установлен и запущен PostgreSQL.

Создайте базу данных `webdev_db` и пользователя `user` с паролем `password` (или измените эти данные в `backend/backend/settings.py` на свои).

**Пример создания базы данных через `psql`:**

```bash
sudo -u postgres psql
CREATE USER user WITH PASSWORD 'password';
CREATE DATABASE webdev_db OWNER user;
\q
```

### 5. Применение миграций

Примените миграции Django для создания таблиц в базе данных.

```bash
python manage.py migrate
```

### 6. Создание суперпользователя

Создайте учетную запись суперпользователя для доступа к административной панели Django.

```bash
python manage.py createsuperuser
```

### 7. Запуск сервера разработки

Запустите локальный сервер Django.

```bash
python manage.py runserver
```

Сервер будет доступен по адресу `http://127.0.0.1:8000/`. Административная панель доступна по `http://127.0.0.1:8000/admin/`.

## API Эндпоинты

Основные эндпоинты будут доступны по префиксу `/api/`.

*   `/api/users/`: Управление пользователями.
*   `/api/rooms/`: Управление комнатами.
*   `/api/schedule/`: Управление расписанием.
*   `/api/university/`: Управление данными университета.
*   `/api/token/`: Получение JWT токенов.
*   `/api/token/refresh/`: Обновление JWT токенов.
*   `/api/token/blacklist/`: Отзыв JWT токенов.
*   `/ws/room/<room_name>/`: WebSocket эндпоинт для голосового чата.
